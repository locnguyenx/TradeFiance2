# Journal Entry: 2026-04-22 - Playwright & Next.js Hardening Lessons

## Critical Issue: Persistent E2E "Stuck" State
**Symptom**: `npx playwright test` hangs for 5-10 minutes during the `webServer` check, failing with `ECONNREFUSED` or timeout.

### Root Cause 1: DNS Dual-Stack Ambiguity
In modern macOS environments (Node 17+), `localhost` resolves to both `::1` (IPv6) and `127.0.0.1` (IPv4). 
- Next.js production mode (`next start`) often binds to IPv4 by default or picks the first available.
- Playwright's `webServer` block might attempt to verify the other address family, leading to a silent polling hang.
- **Fix**: Explicitly use `127.0.0.1:3000` for both `baseURL` and `webServer.url`.

### Root Cause 2: Zombie Turbopack Proxies
`next dev --turbo` spawns multiple child processes. When Playwright terminates the managed `webServer` process group, the Turbopack workers sometimes survive as orphans holding port 3000.
- **Fix**: Use `npm run build && npm start` for E2E verification. The single production process is much more stable and SIGTERM-compliant.

### Root Cause 3: Path-Based Readiness Gates
Polling the root `/` can sometimes succeed (200 OK) before the Next.js hydration engine or internal data fetching is interactive.
- **Fix**: Set the `webServer.url` to a functional business route (e.g. `/import-lc`) to ensure the server is 100% interactive before tests start.

## Implementation Standard
- Port: `3000` (Standard)
- Binding: `127.0.0.1`
- Mode: `Production` (via `npm start`)
- Timeout: `300s` (to allow for build phase latency in constrained environments).
